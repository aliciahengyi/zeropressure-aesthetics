# 零压审美

一套**极简、单色、安静**的整体美学思路。三个字概括：**极简，极简，极简。**

它不是某个工具的规范，而是一种**看待"视觉呈现"的态度**——无论做的是一个 App 界面、一张海报、一套幻灯片、一份文档还是一张图表，都可以按它来。

内核只有一句：

> **让人安心，而不是让人紧张。** 每一个颜色、每一条线、每一个元素，都要服务于"安静"，而不是抢眼、催促或评判。能不加的就不加。

## 一、配色是地基（最重要）

**只用一种颜色**，靠这一种颜色的**深浅**区分主次，绝不靠多种颜色堆叠。

- 最重要的元素颜色最深；次要的浅一些；背景近乎白。
- 该突出的、次要的、背景，全是同一种颜色的不同深浅。

六套可直接用的莫兰迪色板，每套用**两个主色**命名（`bg`=背景近白，`accent`=重点色，`ink`=正文深色，其余为由浅到深的层次色）：

| 名称（两主色） | bg | accent | ink | 层次色 |
|---|---|---|---|---|
| 苔绿 × 燕麦 | `#F1EFE8` | `#6E8C78` | `#4B473E` | `#8BA995` `#A1C1A1` `#C9BBA6` `#C2CDB8` |
| 雾蓝 × 米杏 | `#F1EEE7` | `#7E96B8` | `#4A463E` | `#A8BEDF` `#C7D5E8` `#D8C9BA` `#CAD6E4` |
| 桃粉 × 豆绿 | `#F6F1ED` | `#C77E84` | `#4D4642` | `#E6A6AC` `#A1C2B1` `#B7D5C6` `#EAC3C2` |
| 藕荷 × 水青 | `#F0EEF1` | `#8478A0` | `#494750` | `#B6A9C8` `#A09BC3` `#9DBFC3` `#C9C2DA` |
| 赭红 × 青碧 | `#EDF0EC` | `#BF6C64` | `#48443F` | `#D48982` `#99B6B4` `#DFB199` `#BACFCE` |
| 驼色 × 杏色 | `#F4EFE7` | `#9E8268` | `#4D4536` | `#CBAF98` `#DBB9A4` `#C9B79E` `#D8C8B2` |

描边 / 分隔线（`line`）不是固定值，用 `ink` 和白色按约 16:84 混合算出——六套算出来都接近 `#E2E2E0`，无需单独记。

用户可选其中一套，也可自组一套同逻辑的新色。"换风格"= 换一整套这样的同色深浅，而不是往里加新颜色。

**对比度提醒**：重点色上放白字要小心，不同色相饱和度不同、对比度要自己核对一次——以"苔绿"为例，`#6E8C78` 配白字约 3.7:1，够大字和按钮，但小号正文不够，要压深到约 `#5E7A68`（≈4.6:1）才达标；换别的主色同理，小字前先查一次对比度。

## 二、默认不用框框

极简的核心动作：**默认不用框框、边框、卡片、底色块把东西圈起来。** 元素平铺，靠留白和层次分隔，不靠画框。

- 默认不给内容套卡片，不给一组选项套外框。
- **卡片不是禁止，只是不是第一选项**——用户明确要卡片就可以做，但默认先不用，能不套就不套。
- 在界面里，**按钮可以带框**——只有"要点/要按的东西"允许有边框，用来暗示"这里可以操作"。

## 三、分区靠"自然分隔"，不靠框

要分出不同区域（页面板块、海报段落、文档章节……）时，按这个优先级递减来选，而不是画框：

1. **首选：一条极细的横线**——可用重点色，也可用描边色。
2. **次选：左边竖排细色条**——标签竖着排在色条旁，内容跟在下面（不是必要，视需要选用）。
3. **再次：小标题用重点色，文字用非重点色**——标题和正文的色差自然分区。
4. **最后才用框**——即使用框，也首选细的圆弧框，非必要不填色。

## 四、重点色用于区分

要突出"重要 / 选中 / 可操作"时，**用重点色（最深的那个色），而不是用框、不用大色块。**

- 选中某项 → 那项的字变重点色。
- 当前所在 / 关键数字 / 想让眼睛先落到的地方 → 上重点色。
- 一句话：**该强调的地方上重点色，其余一律安静。** 重点色因为稀有才有力量，不要滥用。

## 五、控件与元素：小而克制

- 按钮、标签、图标都**做小、做轻**，够看清、够用就行，不要大块大色抢戏。
- 主次用"一个填充重点色 + 一个描边"就足够表达，不要一排抢眼元素堆一起。
- 图形**秀气、忌尖锐**：圆角、细线、克制；小图形（勾选、箭头、开关）都往柔和了做。

这套审美落到一个真实控件上长什么样——一个"细线轨道 + 大圆钮"的开关，比常见的实心大开关秀气得多，可作范例：

```html
<!-- 通用网页版 -->
<div class="lsw on"><div class="lsw-ln"></div><div class="lsw-kb"></div></div>
```

```css
.lsw    { position: relative; width: 40px; height: 20px; }
.lsw-ln { position: absolute; top: 7.5px; left: 2px; width: 36px; height: 5px; border-radius: 2.5px; background: #cfc9bd; }
.lsw.on .lsw-ln { background: #6E8C78; }              /* 开：细轨道变重点色 */
.lsw-kb { position: absolute; top: 0; left: 0; width: 20px; height: 20px; border-radius: 50%; background: #fff; box-shadow: 0 1px 3px rgba(80,70,50,.28); transition: left .2s ease; }
.lsw.on .lsw-kb { left: 20px; }                       /* 开：圆钮滑到右 */
```

精髓：**轨道细、圆钮粗**（钮径是轨道高的约 4 倍），一个圆点轻轻压在细线上滑动。这种"轻"，就是整套审美的缩影。

**控件样式改过一次，就是全局默认，别再问第二次**：任何小控件（编辑笔的朝向、开关的样式……）一旦被用户改样式并确认，立刻当规则记下来，之后所有同类控件默认都长这样。不要每次遇到新的地方又重新问一遍——这条是踩过坑才定的：笔的样式改过好几次、开关也改过好几次，每次都被重新问。

## 六、字体与层级

- **字体**：干净的无衬线（系统默认的苹方 / 思源黑体一类即可），不用花体、装饰体。
- **只分三级**：主标题（页面 / 主题）、分区小标题（用重点色）、正文。层级靠**字号 + 颜色深浅**拉开，不靠一路加粗。
- **字重克制**：整体用常规字重，别忽粗忽细；要强调，用**重点色**而不是加粗。
- **字号偏大一档**：读得轻松，优先于塞得多。
- 一个温度细节：楷体这类"手写感"字体，只留给**用户亲手写下的内容**，系统自己的字不用。

## 七、留白与间距

- **留白慷慨**：宁可空，也不要挤。空白不是浪费，是让人喘气的地方。
- 内容与边缘之间留出从容的边距，不要顶到边。
- 间距有节奏、保持一致，别忽宽忽窄。

## 八、文字与用词

**适用范围**：这一节只用于用户自己设计 UI / 交互、需要给按钮、提示语、标签这类**产品自身文案**取名的场合。如果用户只是拿这套美学做视觉呈现（排版、配色、海报、PPT、文档……），内容是用户自己的文字或引用的原文，**不要擅自改用户的用词**——原样呈现，别按这节的语气去"温和化"或"诗意化"。

语气是**陪伴，不是指挥**。温和、带一点温情与诗意，却能让人一下明白指的是什么。

- **少用命令式**：把"要做""必须""去完成""赶紧"这类命令 / 催促词，换成温和的中性说法——不是发号施令，是并肩。
- **有温情**：用词带一点人的温度，像朋友，不像系统提示。
- **一点诗意，但不牺牲清楚**：可以有意境，但一眼要能懂它指什么；诗意服务于感受和理解，不是让人猜谜。好例子——把"今日 / 想法"叫成「当下」「某刻」，有味道，又立刻明白。
- **不评判**：没做完不是过错。

词汇替换示意（命令 / 生硬 → 温和 / 中性）：

| 别用 | 换成 |
|---|---|
| 今天**要**完成三件事 | 今天，想做的三件事 |
| **立即 / 赶紧**处理 | 等你有空时 |
| **必须**打卡 | 记得的话 |
| 完成**任务** | 做完 / 落定 |

## 九、Do's & Don'ts

| 这样做 | 别这样 |
|---|---|
| 单色，靠深浅分主次 | 多种颜色堆在一起 |
| 靠标题、细线、留白分区 | 到处套卡片和边框 |
| 重点色只点睛（因稀有而有力） | 大面积铺重点色、处处强调 |
| 按钮小而克制 | 一排抢眼的大按钮 |
| 细线、圆角、柔和 | 尖锐、粗重、抢眼 |
| 文字安静、有温情 | 命令式、催促、评判的措辞 |
| 拿不准就减 | 加装饰性的、可有可无的元素 |

## 气质（贯穿始终）

不喧哗、不焦虑。配色、动效、间距、用词，都朝"让人安心地把事放下"去做，而不是催促、评判或制造紧迫。**每加一样东西，先问它是否真的必要——能不加就不加。**
